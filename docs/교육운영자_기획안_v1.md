# Clausion - 교육 운영자(Education Operator) 역할 기획안

**문서 버전**: v1.0  
**작성일**: 2026-04-12  
**기술 스택**: React 19 + TypeScript (Frontend) / Spring Boot 3.3.5 + Java 17 (Backend) / PostgreSQL 16  
**공모전 주제**: AI활용 차세대 교육 솔루션

---

## 1. Executive Summary

현재 Clausion 플랫폼은 **교강사(INSTRUCTOR)**와 **수강생(STUDENT)** 2개 역할만 존재한다. 그러나 실제 교육 현장에서는 **교육 운영자**가 교육 품질 관리, 과정 편성, 수강생/교강사 관리, 성과 분석 등 교육의 전체 라이프사이클을 총괄하는 핵심 역할을 수행한다.

교육 운영자 역할을 추가함으로써 Clausion은 단순 LMS를 넘어 **교육 현장 구성원 전체의 페인 포인트를 해결하는 AI 솔루션**이라는 공모전 가이드라인에 부합하게 된다.

## 2. 교육 운영자 역할 정의

### 2.1 교육 운영자란?

> 교육 운영자(Education Operator)는 교육 과정의 기획, 편성, 운영, 평가의 전 과정을 총괄하며, 교강사와 수강생 간의 교육 활동이 원활히 이루어지도록 지원하는 관리자이다.

### 2.2 핵심 책임 영역

| 영역 | 설명 |
|------|------|
| **과정 운영 관리** | 교육 과정 개설/폐강 승인, 시간표 편성, 정원 관리 |
| **인원 관리** | 교강사 배정, 수강생 반 편성, 출결 관리 |
| **품질 관리** | 교육 만족도 추적, 교강사 평가, 커리큘럼 적정성 검토 |
| **성과 분석** | 전체 과정 KPI 대시보드, AI 인사이트 기반 리포트 |
| **이상 징후 대응** | 중도 이탈 위험군 조기 감지, 교강사-수강생 갈등 중재 |
| **시스템 운영** | 공지사항 관리, 일정 관리, 외부 시스템 연동 |

### 2.3 기존 역할과의 관계

```
┌─────────────────────────────────────────────┐
│              교육 운영자 (OPERATOR)            │
│  - 전체 과정 총괄                              │
│  - 교강사/수강생 관리                           │
│  - AI 리포트 & 의사결정 지원                    │
├──────────────────┬──────────────────────────┤
│   교강사 (INSTRUCTOR)  │    수강생 (STUDENT)     │
│  - 과목 내 교육 활동     │  - 학습 활동            │
│  - 학생 상담/평가        │  - 상담 요청            │
│  - 문제 출제             │  - 스터디그룹 참여       │
└──────────────────┴──────────────────────────┘
```

**핵심 차이점:**
- **교강사**: 자신의 과목 범위 내에서 교육 활동 수행
- **교육 운영자**: 전체 과정을 횡단하여 조율, 분석, 의사결정

## 4. 기술 설계

### 4.1 역할 확장

**User.java Role enum 수정**:
```java
public enum Role {
    STUDENT, INSTRUCTOR, OPERATOR
}
```

**DB 마이그레이션** (V12):
```sql
-- V12__add_operator_role.sql

-- Operator profile table
CREATE TABLE operator_profiles (
    id BIGSERIAL PRIMARY KEY,
    user_id BIGINT REFERENCES users(id) NOT NULL UNIQUE,
    department VARCHAR(100),
    position VARCHAR(100),
    managed_course_ids JSONB DEFAULT '[]',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Announcements
CREATE TABLE announcements (
    id BIGSERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    target_type VARCHAR(20) NOT NULL,
    target_course_id BIGINT REFERENCES courses(id),
    is_urgent BOOLEAN DEFAULT FALSE,
    author_id BIGINT REFERENCES users(id) NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE announcement_reads (
    id BIGSERIAL PRIMARY KEY,
    announcement_id BIGINT REFERENCES announcements(id) NOT NULL,
    user_id BIGINT REFERENCES users(id) NOT NULL,
    read_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(announcement_id, user_id)
);

-- Intervention log for at-risk students
CREATE TABLE intervention_logs (
    id BIGSERIAL PRIMARY KEY,
    student_id BIGINT REFERENCES users(id) NOT NULL,
    operator_id BIGINT REFERENCES users(id) NOT NULL,
    course_id BIGINT REFERENCES courses(id),
    intervention_type VARCHAR(50) NOT NULL,
    description TEXT NOT NULL,
    outcome TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- AI operation reports
CREATE TABLE operation_reports (
    id BIGSERIAL PRIMARY KEY,
    report_type VARCHAR(50) NOT NULL,
    period_start DATE NOT NULL,
    period_end DATE NOT NULL,
    summary_json JSONB NOT NULL,
    anomalies_json JSONB,
    recommendations_json JSONB,
    generated_by VARCHAR(50) NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Completion criteria
CREATE TABLE completion_criteria (
    id BIGSERIAL PRIMARY KEY,
    course_id BIGINT REFERENCES courses(id) NOT NULL UNIQUE,
    min_attendance_rate DECIMAL(5,2) DEFAULT 80.00,
    min_assignment_rate DECIMAL(5,2) DEFAULT 70.00,
    min_score DECIMAL(5,2) DEFAULT 60.00,
    additional_rules_json JSONB,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Satisfaction surveys
CREATE TABLE satisfaction_surveys (
    id BIGSERIAL PRIMARY KEY,
    course_id BIGINT REFERENCES courses(id) NOT NULL,
    student_id BIGINT REFERENCES users(id) NOT NULL,
    overall_score INTEGER NOT NULL CHECK (overall_score BETWEEN 1 AND 5),
    instructor_score INTEGER CHECK (instructor_score BETWEEN 1 AND 5),
    content_score INTEGER CHECK (content_score BETWEEN 1 AND 5),
    nps_score INTEGER CHECK (nps_score BETWEEN 0 AND 10),
    feedback_text TEXT,
    sentiment VARCHAR(20),
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(course_id, student_id)
);
```

## 6. AI 활용 포인트 (공모전 차별화)

교육 운영자 역할에서 AI를 활용하는 핵심 시나리오:

### 6.1 AI 이상 징후 조기 경보 시스템
- Twin 데이터의 급격한 변화 패턴을 감지
- 복수 수강생에서 동시에 성과 하락 시 과정/교강사 레벨 이슈 판별
- 상담 요청 폭주 패턴 감지 → 운영자에게 선제적 알림

### 6.2 AI 주간 운영 브리핑
- 매주 자동 생성되는 운영 인사이트 리포트
- "지난 주 대비 이탈 위험군 3명 증가, 주요 원인은 3주차 과정 난이도"
- 구체적 액션 아이템 제안 (특정 수강생 상담 배정, 커리큘럼 조정 등)

### 6.3 AI 교강사 효과성 분석
- 교강사별 수강생 Twin 개선율 비교
- 상담 후 수강생 성과 변화 추적
- 베스트 프랙티스 자동 추출 및 공유

### 6.4 AI 기반 과정 품질 예측
- 과정 초기 데이터로 최종 수료율 예측
- 조기 개입이 필요한 과정 자동 식별
- 커리큘럼 난이도 적정성 AI 평가

## 7. 기존 시스템과의 통합 포인트

교강사/수강생 기능을 건드리지 않으면서 운영자 기능을 추가하기 위한 통합 전략:

| 기존 시스템 | 통합 방식 | 영향도 |
|------------|----------|--------|
| `User` 엔티티 | Role enum에 OPERATOR 추가 | 최소 (enum 값 추가만) |
| `SecurityConfig` | OPERATOR 역할의 접근 권한 규칙 추가 | 최소 (기존 규칙 유지, 신규 추가) |
| `courses` 테이블 | 기존 테이블 읽기 전용 조회 | 없음 (SELECT만) |
| `student_twins` 테이블 | 집계 쿼리로 히트맵 생성 | 없음 (SELECT만) |
| `consultations` 테이블 | 통계 집계용 조회 | 없음 (SELECT만) |
| `Notification` 시스템 | 공지사항 발송 시 재활용 | 최소 (기존 API 호출) |
| `AnthropicChatService` | AI 리포트 생성 시 재활용 | 없음 (서비스 호출만) |
| `RabbitMQ` | AI 리포트 비동기 생성 | 없음 (큐 추가만) |

**원칙**: 기존 교강사/수강생 코드에 수정 없이, 운영자 전용 컨트롤러와 서비스를 **별도 패키지**로 추가
